# 桌面应用 AI 控制插件 · 开发学习资料

> 本文件夹沉淀了一次**完整、真实**的插件开发全过程：从“给网易云音乐装插件后无法启动”
> 的报障，到最终交付一个**零注入、不改文件、适配启动保护**的网易云控制 MCP 插件。
>
> 这里记录的不只是结论，而是**探索的流程、每一次决策的理由、踩过的坑、以及可复用的
> 方法论和工具**——目的是让后续给**任意**桌面应用开发 AI 控制插件时，能照着复刻。

## 这套资料解决的核心问题

> “我想让 AI 助手控制某个桌面应用（播放器/IM/工具…），怎么从零做出一个可靠的控制插件？”

本次的具体案例是网易云音乐，但**方法论适用于任何 Electron / CEF / Chromium 内核应用**，
其中的“控制层选择决策树”“验证驱动的探索”更是适用于所有桌面应用。

## 一句话方法论

> **控制一个应用，先别想着注入。按“专用 API → 官方协议/调试通道 → 外部自动化 → 代码注入”
> 的顺序，选第一个能用的层。能不改它的文件，就绝不改。每推进一步都立刻实测验证。**
