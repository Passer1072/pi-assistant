# 速查卡 · 30 秒看懂（给桌面应用做 AI 控制插件）

> 一页纸的「秒懂」版。要细节看对应的详细文档（链接在每节末）。

## 一句话方法论
> **控制一个应用，先别想着注入。按「专用 API → 官方协议/调试通道 → 外部自动化 → 代码注入」
> 选第一个能用的层。能不改它的文件，就绝不改。每推进一步立刻实测。**

## 选层决策树（越靠前越好）
```
1 官方/专用 API、现成 MCP        → 有就用
2 官方协议/调试通道              → CEF/Electron: --remote-debugging-port (CDP) ★本项目的解
                                  自定义 URL scheme、系统媒体键/SMTC
3 外部自动化                     → UI Automation / 窗口消息 / 模拟输入
4 代码注入                       → DLL 注入 / 改前端包（最后手段，易撞反篡改/封号）
```
→ 详见 [02-通用方法论](./02-通用方法论.md)

## CEF/Electron 逆向：拿「三把钥匙」
| 钥匙 | 怎么拿（在 App 页面里跑 JS） |
|---|---|
| webpack require | `webpackJsonp.push([[9e4],{1:(m,e,r)=>window.__wpr=r},[[1]]])` |
| Redux/dva store | React fiber 回溯找 `memoizedProps.store` |
| 签名 API 工厂 | 扫「导出 `a`、源码含 url/method/cacheStrategy」的模块 |
→ 复制粘贴级代码见 [03-逆向技术手册](./03-逆向技术手册.md)、脚本见 [toolkit/](./toolkit/)

## 启动目标（CEF/Electron）
```powershell
"app.exe" --remote-debugging-port=9222   # 不改任何文件 → 不触发启动保护
```
连 `http://127.0.0.1:9222/json` 的 page websocket，用 CDP `Runtime.evaluate` 跑 JS。

## 必踩的坑（7 条）
1. 注入 JS **别用长 setInterval**——后台窗口被 Chromium 节流；时序放到外部（Python/Node sleep）。
2. dva effect 是异步 saga，**dispatch 后等 200~600ms 再读** state。
3. 「没反应」先怀疑**缺参**（去读 effect 源码）与时序。
4. 自己构造的对象**字段别给 null**（网易云 item 的 `fromInfo`/`referInfo` 给 null → 播放器崩、推荐页崩）。
5. **缩写名 vs 完整名**：API 返回 `al/ar/dt`，但播放器读 `album/artists/duration`。构造 item 前要
   规范化，否则封面/专辑/歌手图片加载失败（歌能放）。
6. **状态字段可能不可靠甚至颠倒**（网易云 `playingState`：播放=2、暂停=1）。判断播放/暂停别信它，
   **看进度是否推进**（隔 ~650ms 采样两次位置）。
7. 终端中文乱码 ≠ 数据错，按值校验。
→ 全量坑见 [04-避坑与决策记录](./04-避坑与决策记录.md)

## 落地为 MCP 插件
- 复用宿主既有 MCP 约定；写自包含 server；工具分层：**高层语义工具 + 万能逃生口**（调任意 API/派发任意动作/执行任意 JS）。
- Electron 内跑 .mjs：`command=electron.exe` 且 **env 必须加 `ELECTRON_RUN_AS_NODE=1`**；server 指向**仓库内**文件（旁边要有 node_modules）。
- 三重验证：单测 + 类型检查 + 端到端。

## 自演化 MCP（forge）：让 AI 自己造工具
```
没有现成工具 → forge_probe 逆向 → forge_test_tool 试跑 → forge_register_tool 登记
            → 撞【安全门】→ 提示用户去插件页【信任】（一次永久）→ 可用
```
只增不删改；AI 不能删，用户能删锻造工具、删不了内置工具。
→ 详见 [05-自演化MCP框架](./05-自演化MCP框架.md) 与 `mcp-servers/forge/README.md`
