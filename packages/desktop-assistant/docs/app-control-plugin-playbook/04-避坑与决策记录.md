# 避坑清单与决策记录

> 本次真实踩过的坑 + 做过的关键决策与理由。下次遇到类似情况直接对照。

---

## A. 踩过的坑（按出现顺序）

### 1. 一上来想“清理残留” —— 其实目录是干净的
报障是“装插件后无法启动”，但程序目录、注册表（IFEO/AppInit）全干净。
**坑**：被字面报障带偏。**正解**：先只读收证据再定性，结论是“版本不兼容”而非“有残留”。

### 2. PowerShell `-ExecutionPolicy Bypass` 被安全策略拦截
写脚本文件再 `powershell -ExecutionPolicy Bypass -File` 被拒。
**正解**：直接用 PowerShell 工具内联执行命令，不绕过执行策略。能内联就别落脚本+绕策略。

### 3. CDP eval 用 `setInterval` 分多次采样 → websocket 反复超时
最难定位的坑。注入的 JS 里用 `setInterval(...,200)` 采样 10 次本应 2 秒，却超时。
**根因**：网易云窗口在**后台**，Chromium **后台定时器节流**把间隔钳大了。
**正解**：改“**一次 eval 做一件事 + 在 Python/Node 侧 `sleep` + 再 eval 读结果**”。
**通用规则**：注入 JS 里**避免长 setInterval/setTimeout**，时序控制放到外部。

### 4. 派发 `switchResumeOrPause` “没反应”
`dispatch({type:'playing/switchResumeOrPause'})` 状态不变。
**根因**：该 effect 源码读 `payload.triggerScene`，不传 payload ⇒ `undefined.triggerScene`
抛错，被 dva saga 吞掉。**正解**：读源码确认必填参数，补 `{triggerScene:'...'}`。
**通用规则**：“没反应”先怀疑**缺参**与**异步时序**，去读 effect 源码。

### 5. `playingState` 始终是 1/2，pause/resume 看不出变化
后台启动的实例音频设备未真正出声，`playingState` 是恢复的显示态。
**正解**：改用**有 reducer、直接改 state 的可观测动作**（点歌→歌名变；setVolume→音量变）
来验证控制链路，而不是依赖播放态翻转。

### 6. 模块名“骗人”
导出 `play/pause/setVolume` 的模块 681 其实是 **Lottie 动画库**（`version "5.13.0"`）。
**正解**：定位到候选后**读源码 + 实测**确认，别只看名字。

### 7. 终端中文乱码
CDP 返回的歌名/歌手在终端显示成乱码，一度以为数据错。
**正解**：那只是终端编码显示问题，JSON 值是对的——**按 id/字段值校验**，别看终端渲染。

### 8. 生成的 server 复制到别处能否解析依赖
MCP server 会被复制到 `agentDir/software-plugin-assets/`，担心 `@modelcontextprotocol/sdk`
解析不到。**正解**：只要副本仍在仓库树内，Node 会向上找到 `node_modules`；并**实测**了
从子目录运行副本能正常工作。

## B. 关键决策记录

| 决策点 | 选择 | 理由 |
|---|---|---|
| 修复 vs 先诊断 | 先只读诊断 | 报障字面≠根因，避免误删/误改 |
| 联网 vs 凭记忆 | 联网核实 | 目标版本(3.1.32)比知识库新，BetterNCM 兼容性必须查证 |
| 无侵入 vs 侵入 | 用户先选无侵入，后改侵入 | 尊重用户意图；但把“侵入”重新理解为“深度控制内部” |
| DLL 注入 vs CDP | **CDP** | 同等深度，但不改文件 → 不触发 3.1 启动保护，抗更新、可逆 |
| 先做什么实验 | 先验 CDP 端口可用性 | 这是一票否决假设，地基不成立则全盘作废 |
| 动作/参数 来源 | 从源码正则提取 | 猜测不可靠；源码字符串是“真实在用”的铁证 |
| 服务端语言 | Node（非 Python） | 贴合项目既有 MCP 架构；Node24 自带 WebSocket+fetch，零额外依赖 |
| 接入方式 | 改造既有 `software-plugin-manager.ts` | 用户要的是“设置里点安装”；复用既有插件管理器而非另起 |
| 接口定位方式 | 按**特征**而非写死 id | 模块 id/混淆名/action 名会随版本变，特征定位才抗更新 |
| 工具集设计 | 高层工具 + 万能逃生口 | 逃生口(`ncm_api/dispatch/eval`)覆盖未封装功能并利于未来扩展 |

## C. 可迁移的“元经验”

1. **先验证一票否决假设**，再投入。
2. **能不改目标文件就不改**——这条几乎总是对的。
3. **回到真实意图**，别被实现词（“注入器/DLL”）框死。
4. **验证驱动**：每一步在真实运行实例上实测。
5. **从源码取事实，不猜**。
6. **按特征定位接口**，抗版本更新。
7. **时序控制放外部**，注入 JS 保持“一次只做一件事”。
8. **复用宿主既有架构**，落地前先摸清。
